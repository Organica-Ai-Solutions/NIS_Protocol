NIS Protocol Examples

Practical implementations of the Neuro-Inspired System Protocol

Overview

This section contains practical examples of the NIS Protocol in action. Each example demonstrates how the protocol's components work together to create adaptive, context-aware systems for different application domains.

These examples are designed to be educational starting points that you can adapt for your own applications. The complete source code for all examples is available in the examples directory of the GitHub repository.

Toll Booth Simulation

The automated toll booth example demonstrates a complete implementation of the NIS Protocol for processing vehicles at a toll plaza. The system uses multiple agents working together to identify vehicles, verify payment methods, detect anomalies, and manage toll operations.

Key Components:

  • Vision Agent: Processes camera feeds to identify vehicle type, license plate, and occupancy
  • RFID Agent: Detects and processes transponder information for registered vehicles
  • Memory Agent: Maintains records of vehicle history, payment status, and known issues
  • Emotion Agent: Modulates system behavior based on traffic conditions and anomaly detection
  • Reasoning Agent: Makes decisions about toll amounts, lane assignments, and security flags
  • Action Agent: Controls barrier gates, lane signals, and customer information displays

Protocol Integration

This example demonstrates how to integrate NIS Protocol with external AI agent communication protocols, positioning NIS as a meta-protocol that can orchestrate agents across different frameworks.

Key Components:

  • Coordinator Agent: Central hub for message routing and protocol translation
  • MCP Adapter: Connects with Anthropic's Model Context Protocol for tool integration
  • ACP Adapter: Interfaces with IBM's Agent Communication Protocol for standardized agent messaging
  • A2A Adapter: Communicates with Google's Agent2Agent Protocol for cross-platform interoperability

NIS Protocol Features Demonstrated:

  • Protocol translation between different AI agent frameworks
  • Centralized coordination of heterogeneous agent systems
  • Adding cognitive capabilities to external tools and agents
  • Future-proofing applications through protocol abstraction
View Example Details View Code Example

Protocol Integration Code Snippet:

from nis_protocol import CoordinatorAgent
from nis_protocol.adapters import configure_coordinator_agent

# Create coordinator
coordinator = CoordinatorAgent()

# Configure with adapters for external protocols
configure_coordinator_agent(coordinator, config_path="config/protocol_routing.json")

# Process messages from external protocols
mcp_response = coordinator.process({
    "protocol": "mcp",
    "original_message": mcp_message  # Anthropic's Model Context Protocol
})

acp_response = coordinator.process({
    "protocol": "acp",
    "original_message": acp_message  # IBM's Agent Communication Protocol
})

a2a_response = coordinator.process({
    "protocol": "a2a",
    "original_message": a2a_message  # Google's Agent2Agent Protocol
})

Building Your Own Examples

To create your own NIS Protocol implementation:

  1. Define Your Agents: Identify which specialized agents you need for your application domain
  2. Configure Emotional Dimensions: Customize emotional dimensions relevant to your system
  3. Set Up Memory Systems: Define which data needs to be stored short-term versus long-term
  4. Establish Communication Flows: Determine how messages will flow between cognitive layers
  5. Integrate External Protocols: Connect with other systems using protocol adapters if needed

For more detailed instructions, refer to the Implementation Guide in the documentation.
