NIS Protocol Architecture

Understanding the biologically-inspired layered cognitive architecture

Cognitive Layers

The NIS Protocol defines a layered cognitive architecture inspired by the structure of the human brain. Each layer builds upon and transforms the output of previous layers, culminating in actions that affect the environment.

This design enables a more natural form of information processing, where raw inputs are progressively refined and enriched with context as they flow through the system, ultimately resulting in informed decisions and adaptive behaviors.

Layer 1: Perception Layer

The Perception Layer is responsible for processing raw sensory inputs, similar to how the human visual cortex processes visual information. This layer serves as the interface between the system and the external world.

Vision Agent

Processes visual information such as images, video streams, or other visual data. It performs tasks like object detection, pattern recognition, and feature extraction.

Input Agent

Handles non-visual inputs such as text, sensor readings, audio, or structured data. It transforms raw inputs into a format suitable for further processing.

Layer 2: Interpretation Layer

The Interpretation Layer contextualizes and encodes the information received from the Perception Layer, similar to how higher-level visual regions in the brain interpret what we see.

Parser Agent

Structures raw data into meaningful formats, identifying relationships between elements and organizing information.

Intent Agent

Determines the purpose or goal behind inputs, inferring user intentions or the meaning of observed behaviors.

Layer 3: Reasoning Layer

The Reasoning Layer is analogous to the prefrontal cortex, integrating information from multiple sources, weighing options, and making decisions.

Cortex Agent

Integrates all information for high-level reasoning, serving as the central decision-making component of the system.

Planning Agent

Develops sequences of actions to achieve goals, considering constraints, resources, and potential outcomes.

Layer 4: Memory Layer

The Memory Layer, inspired by the hippocampus, stores and retrieves information, providing historical context for decision-making.

Memory Agent

Manages retrieval and storage of information, maintaining both short-term and long-term memory systems.

Log Agent

Records events and system states for future reference, creating a historical record of activities and outcomes.

Layer 5: Action Layer

The Action Layer executes decisions made by the Reasoning Layer, translating them into concrete actions in the environment.

Builder Agent

Constructs responses or commands based on decisions from higher layers.

Deployer Agent

Executes actions in the environment, interfacing with external systems or physical actuators.

Layer 6: Learning Layer

The Learning Layer adapts and improves system behavior based on experience, similar to neuroplasticity in the brain.

Learning Agent

Updates models and parameters based on experience, learning from successes and failures.

Optimizer Agent

Tunes system performance, identifying and addressing inefficiencies or areas for improvement.

Layer 7: Coordination Layer

The Coordination Layer orchestrates communication between agents, ensuring smooth operation of the entire system.

Coordinator Agent

Manages message routing and prioritization, ensuring that information flows efficiently between agents.

Information Flow

The NIS Protocol defines a standardized flow of information between agents, ensuring that data is progressively refined and enriched as it passes through the cognitive layers.

This flow includes both forward processing (from perception to action) and feedback loops (from action and memory back to earlier layers), creating a dynamic system that learns and adapts over time.

Core Information Flow

  1. User Input enters the system through perception agents (Vision and Input Agents)
  2. Perception Processing extracts features and structures, updating the Emotional State
  3. Interpretation by the Intent Agent combines and contextualizes data
  4. Memory Integration provides historical context from the Memory Agent
  5. Decision Making by the Cortex Agent synthesizes all information
  6. Action Execution implements decisions in the environment
  7. Feedback Loop logs actions and updates the system for future decisions

Message Structure

Messages in the NIS Protocol follow a consistent format that includes the primary data payload, metadata, confidence scores, emotional state influence, and priority levels. 

{
  "agent_id": "vision",
  "timestamp": 1621435234.567,
  "status": "success",
  "payload": {
    // Primary data specific to the message type
  },
  "metadata": {
    // Additional information about the processing
  },
  "emotional_state": {
    "suspicion": 0.3,
    "urgency": 0.7,
    "confidence": 0.8,
    "interest": 0.5,
    "novelty": 0.2
  }
}

Implementation Example

The following code demonstrates how to implement the base agent structure that all NIS Protocol agents inherit from: 

class NISAgent:
    """Base class for all NIS Protocol agents."""
    
    def __init__(
        self,
        agent_id: str,
        layer: NISLayer,
        description: str
    ):
        self.agent_id = agent_id
        self.layer = layer
        self.description = description
        self.active = True
        
        # Register with the global registry
        NISRegistry().register(self)
    
    def process(self, message: Dict[str, Any]) -> Dict[str, Any]:
        """Process an incoming message."""
        raise NotImplementedError("Subclasses must implement process()")

Each agent in the NIS Protocol inherits from this base class, ensuring consistent behavior and interfaces across the system.
